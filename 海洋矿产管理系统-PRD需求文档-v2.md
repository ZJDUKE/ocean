## 1. 背景与目标

### 1.1 背景

海南省海洋厅作为新成立的主管部门，负责统筹全省海洋矿产资源的勘查、开发与保护工作。海南省周边海域蕴藏着丰富的海洋矿产资源，涵盖**石油天然气、海砂、天然气水合物（可燃冰）、多金属结核、富钴结壳、热液硫化物**等多种类型。然而，当前海洋矿产资源管理面临以下问题：

1. **数据分散**：矿产资源数据分散于自然资源和规划厅、生态环境厅、农业厅等多个部门，缺乏统一归集与管理
2. **流程割裂**：矿产勘查项目、储量数据、矿业权管理缺少信息化手段，依赖线下流程，效率低下
3. **监管盲区**：缺乏对勘查开发活动的实时监督与环境影响评估机制
4. **审批低效**：矿业权审批依赖传统线下流程，跨部门协同效率低、透明度不足
5. **决策困难**：缺乏三维可视化能力，决策者难以直观理解资源分布与储量状态
6. **响应滞后**：缺少统一的应急响应机制应对海洋矿产开发中的灾险情

### 1.2 产品目标

1. **全流程数字化**：覆盖矿产勘查从立项到验收的全生命周期数字化管理
2. **储量精细化管理**：建立标准化储量管理体系，实现动态监控、三维可视化与自动化报表
3. **矿业权在线化**：对接政务服务网，实现矿业权登记、审批、监督全流程在线化
4. **监管智能化**：集成环境监测数据，实现开发活动环境影响评估与灾险情应急管理
5. **一图可视化**：基于二三维GIS技术，实现矿产资源分布、储量、矿业权等信息的综合可视化

### 1.3 成功指标

| 指标 | 目标值 | 衡量方式 |
|------|--------|---------|
| 矿业权审批时效 | 平均审批周期 ≤ 5个工作日 | 系统统计审批流程各节点耗时 |
| 储量数据更新时效 | 储量变动后 ≤ 2个工作日入库 | 系统记录填报→审核→入库时间 |
| 三维模型覆盖率 | 重点矿区三维模型覆盖率 ≥ 80% | 已建模型/应建模型数量比 |
| 环境预警响应 | 异常数据触发预警 ≤ 15分钟 | 监测数据→告警推送时间 |
| 报表自动化率 | 储量台账自动生成率 100% | 自动生成报表/总报表数 |
| 系统在线率 | ≥ 99.5% | 运维监控统计 |

### 功能模块 1.1：矿产分布子模块

#### 3.1.1 功能概述
本模块旨在帮助矿产勘查管理员基于二三维GIS技术直观查看、查询和管理海洋矿产勘查数据，集成展示海底地形地貌、矿产资源分布、地球物理/化学数据、矿业权分布及各类专题图，为勘查决策提供可视化支撑。

#### 3.1.2 入口与触发条件

#### 3.1.7 边界与异常处理

| 异常场景 | 触发条件 | 处理方式 | 反馈给用户 |
|---------|---------|---------|-----------|
| GIS地图加载失败 | WebGL服务或底图服务不可用 | 前端捕获加载错误，展示备用的静态底图 | 页面中央提示"地图服务暂时不可用，已切换为静态模式" |
| 图层数据量过大 | 同时勾选过多高精度图层 | 限制同时激活图层数不超过8个 | 勾选第9个时提示"同时显示的图层过多，请先取消其他图层" |
| 查询无结果 | 查询条件无匹配数据 | 返回空列表 | 弹窗提示"未找到符合条件的结果，请调整查询条件" |
| 浏览器不兼容 | 用户使用不支持WebGL的浏览器 | 检测WebGL兼容性，若不可用则降级显示 | 页面顶部警告条提示"当前浏览器不支持3D地图，建议使用Chrome/Edge" |
| 数据加载超时 | 网络差导致GIS瓦片数据请求超时（>30s） | 自动重试2次，仍失败则展示已加载部分 | 底部提示条"部分地图瓦片加载超时，已展示可用数据" |
| 权限不足 | 用户无矿产分布查看权限 | 后端拦截请求，前端隐藏入口 | 访问时提示"您暂无矿产分布查看权限，请联系管理员" |

#### 3.9.6 状态与流转

| 状态 | 含义 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| 有效 | 矿权在有效期内 | 登记通过审核 | 有效期止<90天 → 即将到期；注销/废止 → 对应状态 |
| 即将到期 | 有效期剩余不足90天 | 有效期止-当前日期 ≤ 90天 | 办理延续 → 有效；到期 → 已过期 |
| 已过期 | 有效期已截止 | 有效期止 < 当前日期 | 办理延续 → 有效（回溯） |
| 已注销 | 矿权人主动放弃 | 矿权人申请注销 | 不可逆 |
| 已废止 | 因违规被强行终止 | 行政处罚决定 | 不可逆 |

```
[登记通过] ──> [有效] ──剩余<90天──> [即将到期] ──到期──> [已过期]

                  │                        ├── 办理延续 ──────┘
                  └── 行政处罚 ──> [已废止] └── 办理延续 ──> [有效]
```

| 异常场景 | 触发条件 | 处理方式 | 反馈给用户 |

## 4. 非功能需求

### 4.1 性能需求

| 场景 | 指标 |
|------|------|
| 三维模型加载 | 大型模型（>500MB）加载时间 ≤ 10秒 |
| GIS地图首屏渲染 | ≤ 3秒 |
| 列表查询响应 | ≤ 2秒（95%分位） |
| 报表生成时间 | ≤ 30秒（标准报表） |
| 并发支持 | ≥ 200用户同时在线 |
| 数据同步延迟（政务网对接） | ≤ 1分钟 |
| 监测数据刷新频率 | 实时数据 ≤ 5分钟 |

### 4.2 安全需求

| 安全类别 | 要求 |
|---------|------|
| 统一认证 | 对接智慧海洋数据管理平台统一用户中心，SSO单点登录 |
| 权限管理 | 基于RBAC的细粒度权限控制（菜单级 + 按钮级 + 数据级） |
| 数据加密 | 储量数据、矿业权数据加密存储（AES-256），传输加密（HTTPS/TLS 1.3） |
| 操作审计 | 所有增删改操作记录操作日志（操作人、时间、IP、变更内容），保存 ≥ 3年 |
| 防篡改 | 关键状态变更（如矿权注销）需二次确认 + 短信验证码 |

### 4.3 兼容性需求

- 浏览器：Chrome 90+ / Edge 90+ / Firefox 88+ / Safari 14+
- GIS引擎：支持 WebGL 2.0
- 数据导入格式：Excel (.xlsx/.xls) / CSV / SHP / GeoJSON
- 报表导出格式：PDF / Excel (.xlsx)
